从书籍文件中提炼知识，生成可复用的规则/技能。

## 工作流
0. 前置判断 → 1. 读取分析 → 2. 提炼核心 → 3. 生成文件 → 4. 质量验证 → 5. 测试 → 6. 交付

## 书籍分类
- A类(操作知识)：技术书/方法论 → 工具手册/操作规范/实战+顾问
- B类(思维模型)：小说/传记 → 决策框架/响应协议

## 规则生成结构
每个生成的规则需包含：Overview、Core Philosophy、CRITICAL PRINCIPLES、When to Use/NOT、类型专属章节

## 核心原则
1. 不编造命令/API 2. 不依赖网络搜索 3. 长书分块 4. 每步验证

## 格式支持
.md/.txt/.epub/.pdf

## 示例
《1984》→ doublethink-detection 规则：识别信息控制与叙事操纵
